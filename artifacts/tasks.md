# WFTI Vue3 H5 任务拆解

## 前端

### FE-01 工程骨架
- 目标：搭建 Vue3 H5 工程骨架、路由、状态管理、全局样式 token
- 产出：基础目录、路由外壳、状态容器、设计变量、异常页骨架
- 依赖：`artifacts/technical-design.md`、`artifacts/design.md`

### FE-02 题库与人格静态配置
- 目标：将 16 题、人格向量、隐藏人格、人格文案结构化为前端静态配置
- 产出：questionBank、personaMeta、关系链元数据
- 依赖：`FE-01`、`artifacts/specs/question-algorithm-spec.md`

### FE-03 人格算法引擎
- 目标：实现本地计算引擎与单测
- 产出：score mapper、hidden persona detector、distance matcher、测试用例
- 依赖：`FE-02`

### FE-04 落地页与活动状态页
- 目标：实现 landing、已测入口态、未开始/已结束状态页
- 产出：landing UI、CTA、防重点击、最近结果分支、登录占位入口
- 依赖：`FE-01`、`FE-09`

### FE-05 答题主链路
- 目标：实现 16 题逐题答题、进度条、自动切题、回退、断点续答
- 产出：quiz 页面、切题动画、resume 弹层、本地草稿
- 依赖：`FE-01`、`FE-02`

### FE-06 计算过渡页与结果页
- 目标：实现 2.5 秒过渡页、结果页渲染与本地最近结果闭环
- 产出：calculating、result 页面、本地最近结果覆盖逻辑
- 依赖：`FE-03`、`FE-05`

### FE-07 雷达图与关系链组件
- 目标：实现 8 维雷达图、球星对照 / 天敌 / 拍档组件
- 产出：radar chart、relation cards、标签云
- 依赖：`FE-06`

### FE-08 海报生成与占位交互
- 目标：实现海报预览，以及分享名片 / 保存相册 / 站内跳转的占位入口
- 产出：poster template、html2canvas/canvas 方案、失败降级、`待开发` toast
- 依赖：`FE-06`

### FE-09 本地仓储与占位 adapter
- 目标：对本地缓存与占位交互进行统一收口
- 产出：localRepository、placeholder action adapter、web mock 环境
- 依赖：`FE-01`

### FE-10 埋点接入
- 目标：接入页面、答题、结果、海报与 bridge 行为埋点
- 产出：tracker service、事件字典、错误日志钩子
- 依赖：`FE-04`、`FE-05`、`FE-06`、`FE-08`

## 配置与占位交互确认

### PL-01 登录占位入口
- 目标：保留登录入口并统一提示 `待开发`
- 产出：占位交互规则、toast 文案、点击埋点

### PL-02 分享与保存相册占位入口
- 目标：保留分享 / 保存相册入口并统一提示 `待开发`
- 产出：占位交互规则、toast 文案、失败态说明

### PL-03 活动静态配置确认
- 目标：确认活动状态、分享文案、二维码链接的静态配置基线
- 产出：字段清单、默认值、维护方式

### PL-04 站内跳转占位入口
- 目标：保留结果页入口与二维码相关跳转入口，并统一提示 `待开发`
- 产出：占位交互规则、toast 文案、按钮态说明

## QA / 联调

### QA-01 前端主链路联调
- 目标：验证 landing -> quiz -> calculating -> result -> poster 闭环
- 依赖：`FE-06`、`FE-08`

### QA-02 占位交互验收
- 目标：验证登录、分享、保存图片、路由跳转入口均正确提示 `待开发`
- 依赖：`PL-01`、`PL-02`、`PL-04`

### QA-03 设计与多机型回归
- 目标：验证长屏布局、安全区、底栏固定、雷达图与海报表现
- 依赖：核心前端功能完成

### QA-04 边界验收
- 目标：覆盖未开始/已结束、网络异常、断点续答、保存失败、海报失败、本地缓存异常
- 依赖：全部核心功能完成

## 推荐实施顺序
1. 先做 `FE-01`、`FE-02`、`FE-03`、`FE-09`
2. 再推进 `FE-04`、`FE-05`、`FE-06`、`FE-07`
3. 并行完成 `PL-01`、`PL-02`、`PL-04`
4. 完成 `FE-08`、`FE-10`
5. 最后执行 `QA-01` 到 `QA-04`
