# 用户原始需求简报

## 项目
- 标题：世界杯球迷人格大鉴赏（WFTI）H5 活动
- 目标端：手机端 H5
- 前端技术要求：Vue 3
- 视觉基调：白色背景，主题色 `#0082FF`，整体符合世界杯氛围

## 背景
- 时间：2026 世界杯赛期（6~7 月）
- 目标：直播吧需要一个高参与度 H5 互动活动承接赛事流量峰值，提升站内用户活跃度

## 核心页面 / 状态
1. 活动落地页：活动介绍 + 开始测试入口
2. 答题页：16 道题逐题展示，含进度条
3. 结果计算过渡页：答题完成后展示 2.5 秒计算动效
4. 结果页：人格展示 + 雷达图 + 分享入口
5. 分享海报：一键生成的人格海报图片

## 用户流程
落地页 -> 答题页（16 题） -> 过渡页 -> 结果页 -> 分享海报

## 功能要点摘要
### 1. 落地页
- 标题：`WFTI — 世界杯球迷人格大鉴赏`
- Slogan：`你不是在看世界杯，你是在演你自己。`
- 说明文案：`16 道题，3 分钟，测出你的世界杯球迷人格密码`
- 固定底部主 CTA：`开始鉴赏`
- 已测试用户展示：`查看我的结果`，主 CTA 变为 `重新测试`
- 未登录用户点击 CTA 需先拉起 APP 登录弹窗
- 首屏加载支持骨架屏；超过 3 秒展示 loading 动效
- CTA 需防重复点击 0.5 秒
- 不支持下拉刷新

### 2. 答题页
- 固定顺序展示 16 题，不随机
- 每题 3 个选项（A/B/C），分值 1/2/3，仅用于计算不向用户展示
- 顶部进度条：百分比 + `N/16`
- 选项选中高亮 0.3 秒后自动切下一题
- 切题动效：左滑出 + 右滑入，400ms
- 支持回退到上一题，保留已选状态
- 第 16 题结束后自动进入计算过渡页
- 本地缓存中断进度 24 小时，重新进入时提示继续 / 重开

### 3. 结果计算过渡页
- 固定展示 2.5 秒
- 中央展示数据翻滚 / 球场扫描类动效
- 文案：`正在分析你的球迷DNA……`
- 前端完成 8 维评分与人格匹配逻辑

### 4. 结果页
- 结构：人格代号 + 中文昵称 -> 人物插画与标签云 -> 球迷人格解析 -> 球星对照/天敌/拍档 -> 二维码 -> 底部悬浮操作栏
- 展示 4 字母人格代号与中文昵称
- 展示人物形象插画与一句话标签组
- 完整人格解析文案
- 页面支持纵向滚动
- 底部固定悬浮操作：`保存相册` / `分享名片` / `重新预测`
- 结果异步上报后端，与 UID 关联；重复测试覆盖旧结果

### 5. 分享海报
- 前端 Canvas / html2canvas 生成 PNG 海报，宽 750px，高约 1334px
- 海报包含：人格代号、中文昵称、标签、八维雷达图、画像文字、球星对照、二维码、logo
- 二维码跳转活动落地页，带 UTM：`source=poster&personality={代号}`
- 分享渠道：微信好友、朋友圈、微博、QQ、保存图片到相册
- 保存图片需处理相册权限与 toast
- 海报生成中需展示 loading；超过 5 秒提示失败
- 分享成功/取消均回结果页，无额外激励

## 算法规则
### 评分
- 16 题每题得分 1/2/3
- 8 个维度：
  - F1 = 题1 + 题2
  - F2 = 题3 + 题4
  - E1 = 题5 + 题6
  - E2 = 题7 + 题8
  - S1 = 题9 + 题10
  - S2 = 题11 + 题12
  - A1 = 题13 + 题14
  - A2 = 题15 + 题16
- 等级映射：
  - 2~3 -> L -> 1
  - 4 -> M -> 2
  - 5~6 -> H -> 3

### 隐藏人格优先级
- `TROL / 杠精球评家`：题3=A + 题12=C + 题9=A，优先级最高
- `HERO / 四年一醒`：题13=A + 题16=C + 题7=C
- 若同时满足，优先返回 `TROL`

### 向量匹配
- 使用 16 种人格原型向量做曼哈顿距离匹配
- 多人格同距离时按人格表顺序优先
- 前端独立完成计算，不依赖后端接口
- 结果生成后异步上报 UID、人格代号、8 维原始分、时间戳

## 人格数据
- 常规人格：GOAT, FANA, GURU, SOFA, RIOT, MUTE, YOLO, VIBE, SALT, JINX, MOOD, CLIP, CHEF, MONK, SOUL, DRIP
- 隐藏人格：TROL, HERO
- 详细题目、选项、人格文案以用户在群里提供的原始消息为准，后续 proposal/specs 需完整整理入库
