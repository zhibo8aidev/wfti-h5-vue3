# Technical Notes - 世界杯球迷人格大鉴赏（WFTI）H5 活动

## 1. 技术方向
- 前端：Vue 3 / mobile-web / App WebView
- 设计产物：HTML 高保真 mockups
- 人格计算：前端本地完成，不依赖后端接口
- 数据策略：本地草稿缓存 + 本地最近结果缓存

## 2. 前端实现重点
### 2.1 状态管理
- 管理活动状态、登录态、题目索引、答题答案、计算结果、海报生成态
- 本地缓存至少包含：答案数组、当前题索引、最近保存时间、最近结果、版本号
- 草稿缓存有效期 24 小时，过期后自动清空

### 2.2 桥接适配层
需通过 adapter 收口以下宿主能力：
- `ensureLogin()`：拉起登录弹窗并返回登录结果
- `share()`：调起宿主分享面板
- `saveImage()`：请求权限并保存图片到相册
- `openRoute()`：站内路由跳转
- `getEnv()`：获取宿主环境、平台信息

### 2.3 人格引擎
- 接收 16 题答案并映射为 1/2/3 分
- 按 F1/F2/E1/E2/S1/S2/A1/A2 聚合维度分数
- 执行隐藏人格优先检测，再执行 16 种常规人格曼哈顿距离匹配
- 结果对象至少包含：`personality_code`、`personality_name`、`raw_scores`、`level_vector`、`answers`、`computed_at`
- 算法模块必须可独立单测

### 2.4 本地结果存储
- `getLatestLocalResult()`：读取本地最近结果
- `saveLatestLocalResult()`：保存本地最近结果
- `clearLatestLocalResult()`：重新测试或主动清空时调用
- 仅保证当前设备 / 当前 WebView 可读，不做跨设备同步

### 2.5 海报生成
- 使用 Canvas / html2canvas 生成 PNG
- 生成时隐藏固定底栏，避免按钮进入截图内容
- 生成超过 5 秒进入失败态并允许重试
- 保存失败或权限拒绝需给出明确 toast

## 3. 静态配置边界
- 题库、人格原型、隐藏人格规则、人格文案全部前端静态内置
- 活动基础配置、二维码目标链接、分享文案也作为前端静态配置维护
- 若未来接入服务端配置，应通过 adapter 替换，不影响当前主链路

## 4. 埋点建议
至少覆盖：
- 活动页曝光 / CTA 点击
- 登录拉起 / 登录成功 / 登录取消
- 每题曝光 / 选项点击 / 回退点击
- 结果计算完成
- 结果页曝光
- 保存相册点击 / 成功 / 失败
- 分享点击 / 成功 / 取消
- 重新预测点击
- 海报生成失败 / bridge 失败 / 本地缓存读取失败

## 5. 测试关注点
- 16 题切题动画、回退逻辑与状态恢复正确性
- 隐藏人格优先级与并列距离排序正确性
- 24 小时缓存过期清理与继续答题弹层
- 本地最近结果覆盖逻辑
- 海报生成兼容性与权限失败提示
- 登录取消、活动未开始 / 已结束、网络异常等边界态

## 6. 当前结论
- 当前项目按前端单体方案推进即可，不需要后端开发
- 技术评审通过后，可直接进入前端实现阶段
- bridge 未确认项继续通过 adapter/mock 收口，不阻塞主链路开发
