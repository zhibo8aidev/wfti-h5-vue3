# Technical Notes - 世界杯球迷人格大鉴赏（WFTI）H5 活动

## 1. 技术方向
- 前端：Vue 3 / mobile-web / App WebView
- 设计产物：HTML 高保真 mockups
- 人格计算：前端本地完成，不依赖在线计算接口
- 后端职责：结果持久化、历史结果查询、二维码与联动配置下发

## 2. 前端实现重点
### 2.1 状态管理
- 管理活动状态、登录态、题目索引、答题答案、计算结果、海报生成态
- 本地缓存需至少包含：答案数组、当前题索引、最近保存时间、版本号
- 缓存有效期 24 小时，过期后自动清空

### 2.2 桥接适配层
需通过 adapter 收口以下宿主能力：
- `ensureLogin()`：拉起登录弹窗并返回登录结果
- `share()`：调起宿主分享面板
- `saveImage()`：请求权限并保存图片到相册
- `openRoute()`：站内路由跳转
- `getEnv()`：获取宿主环境、UID、平台信息

### 2.3 人格引擎
- 接收 16 题答案并映射为 1/2/3 分
- 按 F1/F2/E1/E2/S1/S2/A1/A2 聚合维度分数
- 执行隐藏人格优先检测，再执行 16 种常规人格曼哈顿距离匹配
- 结果对象至少包含：`personality_code`、`personality_name`、`raw_scores`、`level_vector`、`answers`、`computed_at`
- 算法模块必须可独立单测

### 2.4 海报生成
- 使用 Canvas / html2canvas 生成 PNG
- 生成时隐藏固定底栏，避免按钮进入截图内容
- 生成超过 5 秒进入失败态并允许重试
- 保存失败或权限拒绝需给出明确 toast

## 3. 服务接口语义边界
### 3.1 历史结果查询
- 语义：按 UID 返回最新一次人格结果
- 关键字段：人格代号、中文昵称、原始维度分、更新时间

### 3.2 结果上报 / 覆盖
- 语义：接收 UID、人格代号、答案、原始分、等级向量、时间戳
- 同一 UID 重测应覆盖旧结果
- 上报失败不阻塞结果页展示，允许本地排队重试

### 3.3 活动配置 / 二维码配置
- 语义：下发活动状态、分享文案、二维码 landing 链接、海报 schema、站内路由信息
- 二维码目标默认指向活动落地页，并带 poster UTM 参数

### 3.4 联动消费边界
- 输出最小可消费字段：UID、人格代号、中文昵称、更新时间、维度摘要
- 用于专题、社区、竞猜、个人主页等站内场景消费

## 4. 数据与缓存建议
- 本地缓存 key 建议包含活动标识与版本号，避免历史冲突
- 结果本地暂存应与上报队列分离，避免覆盖用户当前展示结果
- 活动状态、二维码配置建议具备本地短时缓存策略，降低重复请求

## 5. 埋点建议
至少覆盖：
- 活动页曝光 / CTA 点击
- 登录拉起 / 登录成功 / 登录取消
- 每题曝光 / 选项点击 / 回退点击
- 结果计算完成
- 结果页曝光
- 保存相册点击 / 成功 / 失败
- 分享点击 / 成功 / 取消
- 重新预测点击
- 接口失败、桥接失败、海报生成失败

## 6. 测试关注点
- 16 题切题动画、回退逻辑与状态恢复正确性
- 隐藏人格优先级与并列距离排序正确性
- 24 小时缓存过期清理与继续答题弹层
- 海报生成兼容性与权限失败提示
- 登录取消、活动未开始 / 已结束、网络异常等边界态
- 结果上报失败重试不影响结果页展示
