# Product Spec - 世界杯球迷人格大鉴赏（WFTI）H5 活动

## 1. 产品概述
一个嵌入直播吧 App WebView 的手机端 Vue3 H5 活动。用户通过 16 道固定顺序题目获得世界杯球迷人格结果，并查看可分享的结果页与海报。本期采用纯前端实现，题库、算法、结果文案与活动基础配置都由前端静态配置承载，不建设后端服务。

## 2. 业务目标
- 承接 2026 世界杯赛期流量峰值，提升站内互动参与度
- 用人格结果与分享海报增强传播性与讨论度
- 用纯前端方案加快活动上线节奏，减少依赖链路

## 3. 核心用户流程
1. 进入活动落地页
2. 点击“开始鉴赏”，未登录时可按宿主规则先拉起登录
3. 完成 16 道题答题
4. 进入固定 2.5 秒结果计算过渡页
5. 查看人格结果页
6. 生成海报并分享 / 保存，或重新测试

## 4. 页面与模块需求
### 4.1 活动落地页
- 标题：`WFTI — 世界杯球迷人格大鉴赏`
- Slogan：`你不是在看世界杯，你是在演你自己。`
- 说明文案：`16 道题，3 分钟，测出你的世界杯球迷人格密码`
- 固定底部主 CTA：`开始鉴赏`
- 若本地存在最近结果，额外展示 `查看我的结果`，主 CTA 文案切换为 `重新测试`
- 页面支持骨架屏，加载超过 3 秒展示 loading 动效
- CTA 点击后置灰 0.5 秒防重复点击
- 不支持下拉刷新

### 4.2 答题页
- 共 16 题，固定顺序，不支持打乱
- 每题展示：题号、题干、A/B/C 三个选项
- 顶部展示进度条，包含百分比和 `N/16`
- 选中选项后高亮 0.3 秒，再自动切入下一题
- 切题动效为当前题左滑出、新题右滑入，动画时长 400ms
- 支持返回上一题并保留已选状态
- 第 16 题完成后自动进入计算过渡页
- 答题进度使用本地缓存保留 24 小时，重进提示继续或重开
- 异常处理：
  - 网络断开时提示 `网络异常，请检查网络后重试`，已答内容本地保留
  - 本地缓存读取失败时静默从第 1 题重新开始

### 4.3 结果计算过渡页
- 固定展示 2.5 秒，不可跳过
- 中心展示数据翻滚 / 球场扫描类动效
- 底部文案：`正在分析你的球迷DNA……`
- 前端在本阶段完成评分、隐藏人格检测、原型向量匹配

### 4.4 结果页
- 从上到下结构：人格代号 + 中文昵称 -> 人物插画区（含标签云） -> 球迷人格解析 -> 球星对照 / 天敌 / 拍档 -> 二维码与品牌区 -> 底部固定操作栏
- 支持纵向滚动
- 底部固定操作：`保存相册`、`分享名片`、`重新预测`
- 结果基于前端本地计算得到，并写入本地最近结果缓存
- 天敌与拍档不可点击跳转

### 4.5 分享海报
- 使用前端 Canvas / html2canvas 生成 PNG 长图
- 截取结果页内容区，自动隐藏底部操作栏
- 海报包含：人格代号、中文昵称、一句话标签、八维雷达图、画像文案、球星对照、二维码、logo
- 二维码跳转活动落地页，并带 `source=poster&personality={代号}`
- 支持渠道：微信好友、微信朋友圈、微博、QQ、保存图片到相册
- 生成中展示 loading 蒙层；超过 5 秒提示失败并关闭蒙层
- 分享成功 / 取消均回到结果页，无额外奖励反馈

## 5. 结果与缓存需求
- 结果仅保存在前端本地缓存中，不做后端持久化
- 落地页 `查看我的结果` 只面向当前设备 / 当前 WebView 内的最近结果
- 重新测试会覆盖本地最近结果
- 断点续答与最近结果缓存应分开存储，避免互相覆盖

## 6. 设计与视觉约束
- 设计目标为 mobile-web，而非原生客户端
- 视觉主基调：白色背景 + `#0082FF`
- 氛围要求：世界杯赛事感、明快、热血、适合社交传播
- 需预留雷达图、标签云、人物插画与大面积分享视觉空间

## 7. 验收基线
- 用户可以从落地页完整走通 16 题 -> 过渡页 -> 结果页 -> 海报链路
- 答题中断后 24 小时内可恢复进度
- 计算逻辑与结果排序严格遵循题库与算法 spec
- 结果页可在无后端情况下独立展示与本地留存
- 海报生成失败不会阻断结果页主链路
